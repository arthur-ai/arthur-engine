# Arthur Insights — Step 3: Simulate Fake Traces

**Goal:** Generate the traces this application *would* send in production — without running
it. From the black-box scan, author realistic scenarios (input, intermediate steps, output),
encode each as an OpenInference span tree, and send them to Arthur Engine over OTLP.

## Read State

```bash
cat .arthur-engine.env 2>/dev/null || echo "(no state file)"
cat .arthur/insights/scan.json | head -c 4000
```

Parse `ARTHUR_ENGINE_URL`, `ARTHUR_API_KEY`, `ARTHUR_TASK_ID`. If `scan.json` is missing, run
the `arthur-insights-scan` skill first.

---

## Design Scenarios

Author **6–10 scenarios** grounded in the scan. Cover these categories (skip a category only
if it doesn't apply to this app):

| Category | What to simulate |
|---|---|
| Happy path (×2–3) | Typical user requests the app was clearly built for, exercising different tools/branches |
| Edge case | Valid but unusual input: very long, multilingual, boundary values for tool parameters |
| Ambiguous | Input that could plausibly route to more than one tool/branch |
| Tool failure | A tool returns an error or empty result mid-trace (use the scan's error_handling notes) |
| Adversarial | Prompt-injection or scope-escape attempt against the system prompt's constraints |
| Off-topic | Request outside the app's domain |

**Critical — vary output quality.** The point of running evals over these traces is to get
signal. If every simulated output is perfect, the evals prove nothing. Make most outputs
realistic-good, but include 2–3 with *plausible* flaws the real app could produce given its
prompts and logic — e.g., an answer not supported by the retrieved documents, a missed or
wrong tool call, ignoring a formatting instruction from the system prompt, an over-confident
answer to the ambiguous input. Derive flaws from the scan's `risk_surface`; never invent
failure modes the architecture couldn't actually produce.

For each scenario record: `id` (kebab-case), `category`, `description`, `user_input`, the
step-by-step trace plan (which spans: LLM calls with exact messages, tool calls with
args/results, retrievals with documents), `final_output`, and `expected_quality`
("good" | "flawed: <what's wrong>").

Show the scenario list (id, category, one-line description, expected_quality) and **ask the
user to approve before sending anything to the engine**. Incorporate their edits. Then write
the full scenarios to `.arthur/insights/scenarios.json`.

## Build the Trace Sender

Write `.arthur/insights/send_traces.py`. Use the template below, filling in one
`emit_<scenario_id>()` function per scenario that mirrors the app's real span structure from
the scan (root CHAIN → AGENT/LLM/TOOL/RETRIEVER children). Follow OpenInference conventions —
the same attribute reference used by `arthur-onboard-instrument`:

- Every span: `openinference.span.kind`, `input.value`, `output.value`
- LLM spans: `llm.system`, `llm.model_name` (use the scan's detected models),
  `llm.input_messages.{i}.message.role/.content` (the *actual rendered prompts* from the scan,
  with scenario values substituted), `llm.output_messages.0.message.*`, realistic
  `llm.token_count.prompt/completion/total`, and `llm.output_messages.0.message.tool_calls.{j}.tool_call.*`
  when the LLM decides to call a tool
- Tool spans: `tool.name`, `tool.description`, JSON `input.value`/`output.value`
- Retriever spans: `retrieval.documents.{i}.document.content/.score` and a JSON `output.value`

```python
"""Send simulated OpenInference traces to Arthur Engine. Generated by arthur-insights."""
import json
import os
import time

from opentelemetry import trace
from opentelemetry.exporter.otlp.proto.http.trace_exporter import OTLPSpanExporter
from opentelemetry.sdk.resources import Resource
from opentelemetry.sdk.trace import TracerProvider
from opentelemetry.sdk.trace.export import BatchSpanProcessor

ENGINE_URL = os.environ["ARTHUR_ENGINE_URL"]
API_KEY = os.environ["ARTHUR_API_KEY"]
TASK_ID = os.environ["ARTHUR_TASK_ID"]
SESSION_ID = f"simulated-{int(time.time())}"

provider = TracerProvider(
    resource=Resource.create({
        "service.name": "<app_name>-simulated",
        "arthur.task": TASK_ID,
    })
)
provider.add_span_processor(
    BatchSpanProcessor(
        OTLPSpanExporter(
            endpoint=f"{ENGINE_URL}/api/v1/traces",
            headers={"Authorization": f"Bearer {API_KEY}"},
        )
    )
)
trace.set_tracer_provider(provider)
tracer = trace.get_tracer("arthur-insights-simulator")

NS = 1_000_000_000
results = {}


def common(span, scenario_id):
    span.set_attribute("session.id", SESSION_ID)
    span.set_attribute("tag.tags", ["simulated", "arthur-insights"])
    span.set_attribute("metadata", json.dumps({"arthur.simulated": "true", "scenario": scenario_id}))


def emit_<scenario_id>():
    t0 = time.time_ns()
    root = tracer.start_span("<root_span_name>", start_time=t0)
    root.set_attribute("openinference.span.kind", "CHAIN")
    root.set_attribute("input.value", "<user_input>")
    common(root, "<scenario_id>")
    ctx = trace.set_span_in_context(root)

    # LLM span — realistic duration (LLM calls: 800–3000ms; tools: 50–500ms)
    llm = tracer.start_span("<llm_span_name>", context=ctx, start_time=t0 + int(0.01 * NS))
    llm.set_attribute("openinference.span.kind", "LLM")
    llm.set_attribute("llm.system", "<provider>")
    llm.set_attribute("llm.model_name", "<model>")
    llm.set_attribute("llm.input_messages.0.message.role", "system")
    llm.set_attribute("llm.input_messages.0.message.content", "<rendered system prompt>")
    llm.set_attribute("llm.input_messages.1.message.role", "user")
    llm.set_attribute("llm.input_messages.1.message.content", "<user_input>")
    llm.set_attribute("llm.output_messages.0.message.role", "assistant")
    llm.set_attribute("llm.output_messages.0.message.content", "<assistant output>")
    llm.set_attribute("llm.token_count.prompt", 412)
    llm.set_attribute("llm.token_count.completion", 96)
    llm.set_attribute("llm.token_count.total", 508)
    llm.set_attribute("input.value", "<user_input>")
    llm.set_attribute("output.value", "<assistant output>")
    common(llm, "<scenario_id>")
    llm.end(end_time=t0 + int(1.4 * NS))

    # ... TOOL / RETRIEVER / follow-up LLM spans per the scenario's trace plan ...

    root.set_attribute("output.value", "<final_output>")
    root.end(end_time=t0 + int(1.5 * NS))
    results["<scenario_id>"] = format(root.get_span_context().trace_id, "032x")


emit_<scenario_id>()
# ... call each emit function ...

provider.shutdown()  # flushes all spans
print(json.dumps(results, indent=2))
```

Template rules:
- One root span per scenario = one trace; children chain via `context=` so parentage is correct.
- Give every span explicit `start_time`/`end(end_time=)` nanosecond timestamps for realistic
  durations; child span times must nest inside the parent's.
- For the tool-failure scenario, set the tool span's `output.value` to the error payload and
  `span.set_status(trace.StatusCode.ERROR, "<message>")`.
- The script prints `{scenario_id: trace_id}` on success.

## Send and Verify

```bash
python3 -m venv /tmp/arthur-insights-venv 2>/dev/null
/tmp/arthur-insights-venv/bin/pip install -q opentelemetry-sdk opentelemetry-exporter-otlp-proto-http

source .arthur-engine.env
/tmp/arthur-insights-venv/bin/python .arthur/insights/send_traces.py | tee /tmp/arthur-sim-trace-ids.json
```

Merge the printed trace IDs into `.arthur/insights/scenarios.json` (add a `"trace_id"` field
to each scenario), then verify ingestion (spans are processed asynchronously — poll):

```bash
source .arthur-engine.env
FIRST_ID=$(python3 -c "import json; print(list(json.load(open('/tmp/arthur-sim-trace-ids.json')).values())[0])")
for i in $(seq 1 10); do
  FOUND=$(curl -s -H "Authorization: Bearer $ARTHUR_API_KEY" \
    "$ARTHUR_ENGINE_URL/api/v1/traces/$FIRST_ID" | \
    python3 -c "import sys,json; d=json.load(sys.stdin); print('yes' if d.get('trace_id') or d.get('root_spans') or d.get('spans') else 'no')" 2>/dev/null || echo no)
  [ "$FOUND" = "yes" ] && { echo "TRACES_INGESTED"; break; }
  echo "Waiting for ingestion... attempt $i/10"; sleep 3
done
```

**Ingested:** report the scenario → trace_id table and the session ID so the user can view
them in the Arthur UI. **Not ingested after 30s:** check the engine logs / auth, offer to
retry. This step must succeed before Step 4 is meaningful, but the eval step can still run
from `scenarios.json` alone — note that if you proceed.
